git-ի հետ աշխատելու պարզ ուղեցույց. Ոչինչ բարդ չէ ;)

Տեղակայում (setup)

Ներբեռնել git֊ը OSX֊ի համար

Ներբեռնել git֊ը Windows֊ի համար

Ներբեռնել git֊ը Linux֊ի համար

Նոր պահոցի (repository) ստեղծում

Նոր git պահոց ստեղծելու համար, անհրաժեշտ է բացել այն պանակը, որտեղ ցանկանում եք տեղադրել և իրագործել հետևյալ հրամաանը
git init

Ստանալ պահոց (repository)

Պահոցի լոկալ աշխատանքային պատճե կարող ենք ստեղծել հետևյալ հրամանով՝
git clone /պահոցի/հասցե
Հեռավոր սերվերի օգտագործման դեպքում հրամանը կլինի
git clone username@host:/պահոցի/հասցե

Աշխատանքային պրոցես

Ձեր լոկալ git պահոցը բաղկացած է երեք «մասից» Աշխատանքային կատալոգ (Working Directory) պարունակում է բոլոր աշխատանքային ֆայլերը. Ինդեքս (Index) կամ «պատրաստի» ֆայլերի տարածք (Staging Area), պարունակում է տեղեկատվություն այն մասին, թե ինչ ֆայլեր ենք commit անելու և HEAD ցույց է տալիս ձեր վերջին արված commit֊ը.

Նախապատրաստում և commit

Որպեսզի նախապատրաստենք փոփոխությունները (ավելացնենք նրանց Индекс-ում) կօգտագործենք
git add <ֆայլի անուն>
git add *
Սա աշխատանքային պրոցեսի առաջին քայլն է. Կատարված փոփոխությունները կարելի է commit անել
git commit -m "commit֊ի նկարագրություն " հրամանով
Փոփոխությունները կատարվել են լոկալ պահոցում և դրանց վրա «ցույց է տալիս» HEAD, բայց դեռևս չեն կատարվել հեռավոր պահոցում.

Փոփոխությունների ուղարկում

Որպեսզի կատարեք փոփոխությունները հեռավոր պահոցում, իրագործեք հետևյալ հրամանը
git push origin master
Դուք կարող եք փոփոխել master֊ին ցանկացած ճյուղով (branch), որպեսզի ուղարկեք փոփոխությունները այնտեղ։

Եթե դեռ չեք պատճենել գոյություն ունեցող «պահոցը» (repository) և ցանկանում եք միանալ հեռացոր պահոցի, անհրաժեշտ է իրագործել հետևյալ հրամանը
git remote add origin <սերվերի հասցե>
Հիմա արդեն կարող եք ուղարկել փոփոխությունները հեռավոր «պահոց»

«Ճյուղավորում» (Branching)

Ճյուղերը օգտագործվում են ֆունկցիոնալության զարգացման համար, մնացաց ճյուղերից առանձնացված։ master ճյուղը օգտագործվում է լռությամբ, անմիջապես պահոց ստեղծելու ժամանակ։ Օգտագործեք այլ ճյուղեր զարգացման համար և «կպցրեք» master֊ին, երբ զարգացումն ավարտված է։

"feature_x" անունով ճյուղ սարքելու և նրան միանալու համար անհրաժեշտ է git checkout -b feature_x հրամանը
master֊ին հետ միանալու համար անհրաժեշտ է
git checkout master հրամանը
ճյուղը կարելի է ջնջել
git branch -d feature_x հրամանով
Ճյուղը հասանելի չի նրանց, ովքեր աշխատում են ձեր հետ հեռավոր պահոցի վրա, քանի դեռ այն չեք ուղարկել պահոց
git push origin <ճյուղի անուն>

Թարմացում և միավորում

Ձեր լոկալ պահոցը կարող եք թարմացնել հետևյալ հրամանով՝
git pull
որը «վերցնում է» հեռավոր պահոցի փոփոխությունները և միավորում այն տվյալ պահին ակտիվ ճյուղի հետ
Որպեսզի միավորեք փոփոխությունները այլ ճյուղի հետ (օրինակ master), օգտագործեք հետրյալ
git merge <ճյուղի անուն> հրամանը
Երկու դեպքում էլ git֊ը փորձում է ավտոմատ կերպով կատարել փոփոխությունները։ Ցավոք որ միշտ է այն հնարավոր և արդյունքում կոնֆլիկտ էառաջանում։ Դուք պատասխանատու եք առաջացած կոնֆլիկտ֊ների համար։ Անհրաժեշտ է ձեռքով խմբագրել git֊ի նշված ֆայլերը։ Փոփոխություններից հետո դուք պետք է միավորել իրար հետ ֆայլերը
git add <ֆայլի_անուն>
մինչև միավորելը դուք կարող եք դիտել փոփոխությունները
git diff <ճյուղի_անուն> <մյուս_ճյուղի անուն>

Պիտակներ (tagging)

Խորհուրդ է տրվում օգտագործել պիտակներ, որոնք կապված են տվյալ պահին օգտագործվող վերսիայի հետ։ 1.0.0 անունով պիտակ կարելի է ստեղծել հետևյալ հրամանով` git tag 1.0.0 1b2e1d63ff
1b2e1d63ff սրանք առաջին 10֊ը նիշն են այն ինդենտիֆիկատորի(id), որ կապված է պիտակի հետ. Որպեսզի տեսնեք commit-ների ինդենտիֆիկատորները, օգագործեք
git log հրամանը
Կարելի է օգտագործել ավելի քիչ քանակի սիմվոլներ id֊ի համար, ուղղակի պետք է հաշվել , որ այն միակնն է։

Լոկալ փոփոխությունների փոխարինում

Եթե դուք ինչ֊որ բան այնպես չեք արել, կարող եք փոխարինել լոկալ փոփոխությունները հետևյալ հրամանով
git checkout -- <ֆայլի_անուն>
հրամանը փոխարինում է այն աշխատանքային կատալոգի այն փոփոխությունները փոփոխությունները, որոնք գտնվում են HEAD֊ում։ Փոփոխություններրը, արդեն գտնվում են Index֊ում։

Եթե ցանկանում եք ջնջել ձեր բոլոր լոկալ փոփոխությունները և commit֊ները, ստացեք (fetch) բոլոր փոփոխությունները սերվերից և նշեք լոկալ ճյուղը master, այ այսպես։
git fetch origin
git reset --hard origin/master

tweaks և հարմարավետ հրամաններ

git֊ի ներկառուցված գրաֆիկական ինտերֆեյս
gitk
օգտագործեք գունավոր սիմվոլներ տերմինալում
git config color.ui true
commit֊ը մուտքագրել մի տողով
git config format.pretty oneline
Index֊ին ավելացնելու ինտերակտիվ միջոց
git add -i
